### Podpora za serijske rezervacije

Brez razbijanja obstoječega modela:

**`app/models.py`** – dodal `series_id: VARCHAR (nullable, indexed)` na `Reservation`. NULL = enkratna; vsi zapisi iste serije delijo isti UUID.

**`app/database.py`** – v `init_db()` dodal idempotentno migracijo:
```sql
ALTER TABLE reservations ADD COLUMN IF NOT EXISTS series_id VARCHAR;
CREATE INDEX IF NOT EXISTS ix_reservations_series_id ON reservations (series_id);
```
PostgreSQL ≥ 9.6 / SQLite ≥ 3.35 oboje podpira. Sicer se izpis `[init_db] migracija series_id preskočena: …` zgodi in app teče naprej.

**`app/schemas.py`** – nove sheme:
- `WeeklySeriesCreate{prostor, hour, weekday(0-6), date_from, date_to, razred?, teacher_id, qty?, skip_conflicts}`
- `FullDaySeriesCreate{prostor, date_from, date_to, razred?, teacher_id, qty?, hours?, skip_conflicts}`
- `SeriesResult{series_id, created, skipped[]}`
- `ReservationOut` razširjen s `series_id`.

**`app/routers/rezervacije.py`** – nove rute:
- `POST /api/rezervacije/series/weekly` – admin/vodstvo. Generira termine po formuli "vsak teden isti weekday med from in to", v eni transakciji. `skip_conflicts=false` (privzeto) → 409 z razlogom; `true` → vse proste rezervira, konfliktne preskoči.
- `POST /api/rezervacije/series/full-day` – admin/vodstvo. Za vsak delovni dan v razponu rezervira vse ure (default 0..7) ali specifične `hours`. Vikende preskoči (lahko enostavno odstraniš `if d.weekday() < 5`).
- `GET /api/rezervacije/series/{series_id}` – izpis vseh terminov serije.
- `DELETE /api/rezervacije/series/{series_id}` – pobriše celo serijo (admin/vodstvo ali avtor cele serije).
- `_require_admin_or_vodstvo()` helper za RBAC.
- `_conflict_reason()` helper, ki preverja kapaciteto tablic in unikatnost prostora brez vrženja izjeme (uporabljen pri batch-u).

Ker `tablice` dovoljujejo več soobstoječih rezervacij (do `TABLICE_MAX=28`), je za serijo tablic preverjeno, da skupna količina ne preseže kapacitete na nobenem terminu.

### Skripta za uvoz učiteljev

`scripts/import_teachers.py` + `scripts/requirements.txt`.

Logika:
1. Scrape https://www.tonecufar.si/o-soli/zaposleni/ z requests+BeautifulSoup+lxml. Pravim s testom strani: 8 tabel (VODSTVO, ADMINISTRACIJA, STROKOVNE SLUŽBE, TEHNIČNO OSEBJE, RAZREDNA STOPNJA, PREDMETNA STOPNJA, JUTRANJE VARSTVO IN PODALJŠANO BIVANJE, KABINETI). Stolpci: Telefon | Funkcija | Email | Ime in priimek. Trenutno se nahaja ~79 email naslovov.
2. Za vsakega ujamemo email z regexom, ime/priimek iz zadnjega stolpca; deduplikacija po emailu, prioriteta vloge **vodstvo** > teacher.
3. Generiramo geslo: 7 znakov, garantirano ≥1 mala, ≥1 velika, ≥1 številka. Karakter set izloča zamenljive 0/O, 1/l/I.
4. Login na app kot admin (privzeto `admin / admin123`), session cookie.
5. Za vsakega kličemo `POST /auth/admin/users` z `username=email`, `email=email`, `first_name`, `last_name`, `password`, `role`. Pri "Uporabnik že obstaja" zapiše `exists`, ne reseta gesla.
6. Izpiše CSV `email,ime,priimek,vloga,geslo,tab,status` za nadaljnjo distribucijo.

Uporaba (po tem, ko stoji javni URL):

```bash
pip install -r scripts/requirements.txt
python3 scripts/import_teachers.py \
    --base-url https://solski.ostc.si \
    --admin-user admin \
    --admin-pass 'admin123' \
    --out gesla_ucitelji.csv

# Najprej test brez klicev:
python3 scripts/import_teachers.py --dry-run
# Vključi tudi administracijo + tehnično osebje:
python3 scripts/import_teachers.py --include-all
```

V korenu repozitorija (`/home/admin_os/reservation_app/reverse_proxy.md`). Pokriva:
- DNS pri Cloudflare (prestavitev NS, avtomatski CNAME preko `cloudflared tunnel route dns`).
- Nameščanje `cloudflared` v `~/bin/` brez sudo.
- Login + create tunnel + config.yml (subdomena `solski.ostc.si`).
- Trajno delovanje preko `systemd --user` (alternativa: screen).
- Test, monitoring, logi.
- Posodobitev `BASE_URL` v ConfigMap in restart.
- Varianta B (`ostc.si/solski-app`) preko CF Page Rules / Workers / origin nginx-a.
- Plan B z zrok-om.
- Tabela diagnostike.

Glavna opomba: za pot `/solski-app` brez sudo na origin web serverju je najpreprostejša pot Cloudflare Page Rule s 301 redirectom na `solski.ostc.si`, ali transparent rewrite preko CF Worker-ja. Subdomena je močno priporočena (brez prilagajanja `--root-path`).
