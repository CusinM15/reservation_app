# Odgovor — cleanup serijskih rezervacij

## Kaj je bilo narejeno

### Problem
Po dodajanju gumbov za admin/vodstvo (tedenska in celodnevna serija) so bila v modalih nepotrebna polja: razred, ID učitelja in checkbox za preskakovanje konfliktov.

### Spremembe

**Backend (`app/schemas.py`):**
- Iz `WeeklySeriesCreate` in `FullDaySeriesCreate` odstranjeni:
  - `razred: Optional[str]`
  - `teacher_id: int`
  - `skip_conflicts: bool`
- teacher_id se zdaj samodejno vzame iz trenutno prijavljenega uporabnika (`request.cookies`)

**Backend (`app/routers/rezervacije.py`):**
- `_commit_series` ne sprejema več `razred`, `skip_conflicts`
- Ob konfliktu vedno vrne 409 s seznamom konfliktov (ni več opcije skip_conflicts)
- `create_weekly_series` in `create_full_day_series` uporabljata `user.id` iz `_require_admin_or_vodstvo()`

**Frontend (`app/templates/index.html`):**
- Odstranjena polja iz weekly-series-modal: Razred, ID učitelja, Preskoči konfliktne termine
- Odstranjena polja iz fullday-series-modal: Razred, ID učitelja, Preskoči konfliktne termine
- Posodobljene JS funkcije: `openWeeklySeriesModal`, `submitWeeklySeries`, `openFullDaySeriesModal`, `submitFullDaySeries`

### Deployment info (shranjeno v memory)
- k3s: 2 workerja + 1 master
- Load balancer preko reverse proxy na ostc.si
- FastAPI 2-4 replike na workerju
